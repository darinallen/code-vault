CSS

All the latest styles.

Overview

CSS is used to customize the style and presentation of websites. This includes fonts and colors, but it also applies to the layout and positioning of HTML elements on the page.

Classes, IDs, and other Selectors

  • * will select all HTML elements in a file.
  • You can reach an element that is a child of another element by: div div p { /* Some CSS */ }
  • If you want to reach direct children only, you can do that by: div div > p { /* Some CSS */ }
  • CSS gives priority to more specific selectors. A specific selector will override a general selector.
  • Class: A selector you use when a bunch of elements across a variety of selectors that should all receive the same styling.
  • id: A selector to use when you want to specifically style a single html element.
  • span: A selector that allows you to style one stretch of text or part of an element (I think).
  • text-decoration: Can change the look of links in ‘a’ elements.
  • pseudo-selectors: These allow you to set the styling for elements that may not be indicated in the HTML. For example, a link that is visited vs unvisited vs hovered. These selectors take the form of:
    selector: pseudo-class_selector {
         property: value;
    }
    a:hover {
          color: #cc0000;
          font-weight: bold;
          text-decoration: none;
    }
  • a:link: an unvisited link
  • a:visited: a visited link
  • a:hover: A link you’re hovering over with your mouse
  • first-child: A pseudo-class that applies styling only to the elements that are the first children of their parents.
  • nth-child(x): Same thing, but with the nth child, specified with a number for x.
  • body :nth-child(4) means “find the 4th child of body.
  • body:nth-child(4), without the space, means “find the body tag that is a 4th child.
  • You can resize an image with height and width selectors, and you can make it into a circle with a border-radius of 100%.

Positioning

  • Elements populate the page in what’s known as the CSS Box Model.
  • By default, all HTML elements take up the full width of the page. You can change this with the display property.
  • display {
    • block: This makes the element a block box. It won’t let anything sit next to it on the page. It takes up the full width.
    • inline-block: This makes the element a block box, but will allow other elements to sit next to it on the same line.
    • inline: This makes the element sit on the same line as another element, but without formatting it like a block (e.g. it won’t keep width/height dimensions). It only takes up as much width as it needs to (not the whole line). This parameter is better suited for HTML elements that are blocks by default, such as headers and paragraphs.
    • none: This makes the element and its content disappear from the page entirely.

  • margin: The space around an element. You can adjust the margin to move HTML elements closer or farther away from one another.
    • auto: Automatically puts equal left and right margins on an element, centering it on the page.
    • margin-top, margin-bottom, margin-left, and margin-right add margin to those areas. You can also set these margins all at once, starting at top and going in clockwise order to right, then bottom, then left. e.g. “margin: 1px 2px 3px 4px;”
  • border: The edge of the element. Can be structured as “border: 2px dashed #FF0000;”
  • padding: The spacing between the content and the border. When you give CSS a positive padding or margin value, it puts that space between the element and its reference. You can give padding a negative value, which will move the element in the other direction.
  • content: The actual “stuff” in the box. Example: for a <p> element, the “stuff” is the text of the paragraph.

  • float: Puts an element in the flow of other elements, so floating elements won’t stack on top of one another.
  • clear: Tells an element to clear the other elements on the page. clear: left will immediately move below any floating elements on the left side of the page. clear: right will do the same on the right side. clear: both will get out of the way of elements on the left and right. This works well for footers that need to be placed at the bottom of the page below other elements.
  • position: static: The default positioning if you do not specify otherwise.
  • position: absolute: Positions the element in relation to the first parent element that doesn’t have position: static. If there is no such element, the element with position: absolute gets positioned relative to <html>.
  • position: relative: Tells the element to move relative to where it would have landed if it just had the default static positioning.
  • position: fixed: Anchors an element to the browser window.
  • z-index: A measure of importance. The higher an element’s z-index, the higher it will be stacked on the page. A z-index of 1 when no other elements have a z-index will ensure the element sits on top of other elements and won’t get covered up.
  • CSS now have behavior with features like :hover and animations. It also have structure with pseudo-elements ::before and ::after.
